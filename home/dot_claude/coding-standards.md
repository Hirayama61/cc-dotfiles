# コーディング規約(正典)

このファイルが Claude Code のコーディング規約の**単一ソース**。コード編集時
(Edit/Write/MultiEdit/NotebookEdit)に hook 経由で自動注入される。文面を
他所へ複製しないこと(複製は二重メンテと陳腐化を生む)。

## コメント方針

- コードそのものに意図を語らせる。命名・分割・構造で表現し、コメントで
  補わなくても読める状態を目指す。
- 自明な what コメント(コードを言い換えるだけのもの)は書かない。
- 残してよいのはコードから読み取れないもの限定:
  - 経緯(なぜこの変更/この形になったか)
  - なぜこの実装か(他案を検討して捨てた理由)
  - workaround(苦肉の策とその原因・参照先 issue/PR/URL)
  - 非自明な前提・制約(外部仕様、順序依存、既知の落とし穴)
- 既存コードに合わせる原則と整合させる。周辺がコメント過多でも、それに
  倣って冗長コメントを増やさない(冗長は持ち込まない)。
