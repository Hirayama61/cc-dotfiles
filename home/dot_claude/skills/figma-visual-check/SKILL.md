# figma-visual-check — Figma と実装の視覚照合

Figma 実装ミス(ヘッダー色違い・余白ズレ等)の大半は「Figma MCP の読取値を盲信して
裏取りしない」ことから生まれる。このスキルは**読取値を未検証扱い**とし、
実画面のスクリーンショットとの目視比較で初めて「実装完了」と言えるようにする。

## 不変条件(必読)

- **Figma MCP の返す色・数値・構造は、スクリーンショット照合が済むまで未検証扱い**。
  Figma の内部構成(グループ化・detach・上書き)次第で MCP 読取値は見た目と食い違う。
- 照合を実施できなかった場合は**「照合 skip(理由)」と必ず明示**する。サイレントに
  省略して「完了」と報告しない。
- 差分を「軽微だから」と自己判断で握らない。残す差分は人間の ack を取る。

## 入力

呼び出し側(fe-implement / fe-qa / 人間)から受け取る:

- 対象: Figma の URL(または人間が添付した見本画像)
- 実装側: 対象画面の URL(dev サーバ)と対象領域(コンポーネント/画面全体)
- ビューポート: Figma フレーム幅に合わせる(不明なら 1440px を既定)

## 手順

### 1. 見本(expected)の取得

1. Figma MCP が使えるなら、対象ノードの**画像エクスポート**を取得する(値の読取でなく
   レンダリング画像。これが見た目の正)。
2. 画像エクスポートが取得できない場合は、**人間に見本のスクリーンショット添付を依頼**する
   (AskUserQuestion か依頼文で)。値だけで照合を済ませない。
3. どちらも得られない場合は「照合 skip(見本が取得できない)」と明示して呼び出し側へ戻る。

### 2. 実装(actual)の取得

Playwright MCP で対象画面を開き、見本と同じビューポート幅でスクリーンショットを取る
(`browser_resize` → `browser_navigate` → `browser_take_screenshot`。対象領域が
コンポーネント単位なら要素スクリーンショットを使う)。
Playwright MCP 不在・dev サーバ起動不能なら「照合 skip(理由)」と明示して戻る。

### 3. 項目別比較

見本と実装の画像を**並べて読み込み**、次の項目を1つずつ判定する(一括の「だいたい同じ」禁止):

| 項目 | 見るもの |
|---|---|
| 色 | 背景・文字・ボーダー・状態色(hover 等は対象なら) |
| 余白 | padding / margin / gap の相対関係(視覚的な詰まり・空き) |
| タイポグラフィ | サイズ・weight・行間・truncation |
| 整列 | 揃え位置・グリッド・要素の順序 |
| 形状 | 角丸・影・ボーダー幅・アイコンの種類とサイズ |

判定は `一致` / `差分あり(内容を具体的に)` / `判定不能(理由)` の 3 値で表にする。

### 4. 差分の解消ループ

- `差分あり` は修正 → 手順 2 から再照合。ループは差分ゼロまで回す。
- 修正で解消しない・意図的にデザインから外す差分は、**人間に差分内容を示して ack** を取る
  (ack された差分のみ「意図的差分」として残せる)。

### 5. 報告

```text
## figma-visual-check 結果
対象: {画面/コンポーネント} / 見本: {Figma URL or 添付} / ビューポート: {幅}
| 項目 | 判定 | 詳細 |
|---|---|---|
| 色 | 一致 | - |
| 余白 | 差分あり→修正済 | ヘッダー下 padding 不足を修正(2 周) |
照合できなかった項目・skip した理由があれば必ず明記。意図的差分は ack 済みとして列挙。
```
